Handoff 01 · Executive

Spravio — operação de agência de dev em um só lugar.

Este é o pacote de handoff consolidado das telas internas do produto: gestão de portfólio, projetos, sprints, PRs, forecast, financeiro, integrações, settings e email kit transacional. Alvo: um engenheiro (ou time) implementando o produto de ponta a ponta com Claude Code CLI. Os mockups deste pacote já estão em React+CSS vanilla — basta portar para Next.js + Tailwind/CSS modules.

O que este handoff entrega

  • Design tokens exatos (oklch + hex), prontos pra colar em tokens.css ou tailwind.config.ts.
  • Mockups HTML/React navegáveis de cada tela, com dados fake BR-realistas (spravio_portfolio/data.js).
  • Specs por tela — endpoints, shapes, estados, edge cases, interações-chave.
  • Componentes compartilhados documentados com path, props e variantes.
  • Sitemap visual conectando as telas via navegação real da app.
  • Zip com todos os HTMLs ao final, pronto pra importar num sandbox.

Out of scope neste pacote

  • Landing page (Spravio Landing.html) e seu handoff próprio em design_handoff_spravio_landing/.
  • App mobile (Spravio Mobile Screens.html) — tem sitemap próprio.
  • Tela de auth (Spravio Auth Screens.html) — referência visual, specs não entram aqui.
  • Backend data model final — este doc descreve shapes da UI; o dev deve derivar o schema Postgres/Prisma a partir disso.

Next steps sugeridos

  1. Semana 1: scaffolding (Next.js app router + auth + shell + tokens). Porta shell.css + shell_app.jsx.
  2. Semana 2–3: Projects list + Project detail (as duas telas mais densas). Usar fixtures do data.js como seed.
  3. Semana 4: Portfolio + Forecast (só quando Projects já tem dados reais pra agregar).
  4. Semana 5: Sprints, PRs, Financial (dependem de integrações — começar com mocks).
  5. Semana 6: Integrations + Settings (CRUD puro).
  6. Semana 7: Email kit (MJML ou React Email, templates prontos).
Nota pro Claude Code: este handoff referencia os arquivos HTML no mesmo projeto. Abra cada link de Preview lado-a-lado com seu editor. Os mockups são a fonte da verdade visual; este doc é a fonte da verdade de comportamento + dados.
Handoff 02 · Sitemap

Como as telas se conectam.

Sidebar fixa + topbar compartilhada + conteúdo principal. Todas as rotas sob /app exigem auth.

Regras de navegação

  • Sidebar é persistente: largura fixa 224px, colapsável em xl < 1280px.
  • Topbar tem Command palette (⌘K) que faz fuzzy-search em projetos, devs, PRs, clientes.
  • Botão "Novo projeto" no topbar abre modal stepper de 4 steps — disponível em qualquer rota.
  • Project detail tem 9 abas com ?tab= na URL: overview, sprints, backlog, team, finance, prs, docs, risks, activity.
  • Email kit não é rota da app; são templates renderizados server-side (React Email / MJML) e enviados via provider.

Shape (Project)

type Project = {
  id: string,
  key: string,          // ex: "VIV-2024"
  name: string,
  client: { id, name, sector },
  status: "discovery"|"active"|"paused"|"done",
  health: "green"|"yellow"|"red",
  healthScore: number, // 0-100
  pm: User,
  team: User[],
  contractType: "fixed"|"tm"|"retainer",
  budget: number,     // R$
  budgetConsumedPct: number,
  hoursEst: number,
  hoursLogged: number,
  startDate: Date,
  deadline: Date,
  sprints: Sprint[],
  tags: string[],
}

Interações

  • Click numa row → navega pra /app/projects/[key]?tab=overview.
  • Click na chave (mono) → copia pra clipboard + toast.
  • Hover no BudgetBar → tooltip com R$ consumido / R$ total · X% das horas.
  • Filtros são URL state (?status=active&pm=renata), compartilháveis.
  • Botão "Novo projeto" no topbar abre o <NewProjectModal>.
  • View toggle: tabela (default) ↔ cards (variation_b.jsx).

Estados & edge cases

  • Empty: nenhum projeto → CTA "Criar primeiro projeto" (hero).
  • No results: filtros ativos, 0 rows → "Nenhum projeto corresponde. Limpar filtros".
  • Loading: skeleton rows (6 linhas, shimmer).
  • Error: banner topo "Falha ao carregar — tentar novamente".
  • Row sem PM: mostrar "— sem PM —" em --ink-3.
  • Budget >100%: bar em --bad + chip "over" na coluna status.
  • Deadline passado & status != done: data em --bad + tooltip "Atrasado X dias".
